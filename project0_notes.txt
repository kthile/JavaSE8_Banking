Create a program that simulates banking transactions

- User interaction done with scanner
- Users should be able to register with a username and password
	* MUST be able to apply for joint account
- Users should be able to withdraw, deposit, and transfer funds between accounts
	* Validate for negative amounts, overdrawing, etc.
- Employees should be able to view customer info
	* Account info
	* Account balance
	* Personal info
- Employees should be able to approve or deny applications for accounts
- Bank admins should be able to view and edit ALL accounts
	* Approve/denying accounts
	* Withdrwaing, depositing, transferring from all accounts
	* Cancelling accounts
- Use file i/o to pass info
- 100% test coverage is expected with J-Unit
- Use Log4j to log transactions
